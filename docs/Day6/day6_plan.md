## Day6 学习路径

### 第 1 步：优化页面标题和项目说明

目标：

让页面一打开就能看出这是一个“本地知识库问答 Demo”，而不是普通测试页面。

当前页面已经改为：

```python
st.set_page_config(page_title="我的 RAG 知识库问答 Demo", page_icon="🔎")
st.title("我的 RAG 知识库问答 Demo")
st.write("输入一个问题，系统会从本地知识库检索相关片段，再结合大模型生成带来源参考的回答。")
```

为什么要做：

- 项目展示时，标题和说明会影响第一印象。
- 面试官可以快速看懂 Demo 的用途。
- 这一步能训练把代码功能转化成项目表达的能力。

运行命令：

```powershell
.\.venv\Scripts\python.exe -m streamlit run docs\Day4\streamlit_rag_demo.py
```

预期输出：

- 页面标题显示“我的 RAG 知识库问答 Demo”。
- 页面说明能表达“本地知识库检索 + 大模型回答 + 来源参考”。
- 原有问答、命中片段、来源引用和资料不足提示仍然正常。

常见报错：

- 页面标题没变化：检查是否保存了 `streamlit_rag_demo.py`。
- 页面仍显示旧内容：刷新浏览器或重启 Streamlit。
- 字符串引号报错：检查中文说明外面是否有成对的双引号。

必须掌握的知识点：

- `st.set_page_config()` 控制浏览器标签页标题和图标。
- `st.title()` 控制页面主标题。
- `st.write()` 可以展示项目说明文字。
- 页面文案也是项目展示的一部分。
