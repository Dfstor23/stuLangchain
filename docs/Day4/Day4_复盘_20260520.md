## questions.md：检查题与点评

### 第 1 组：最小 Streamlit 页面

已掌握：

- Streamlit 是展示层，CLI 能跑通后再做页面更稳。
- `st.text_input()` 用来接收用户输入。
- `st.button()` 用来响应用户点击提交。
- `TOP_K` 变小会减少检索片段，回答可能更短或不完整。

纠正点：

- 页面标题原始代码是 `st.title("Day4 RAG Demo")`，后来已改成 `st.title("RAG A&Q")`。

### 第 2 组：命中片段展示

已掌握：

- 展示命中片段可以帮助判断模型回答是否基于资料。
- `result["hits"]` 是检索返回的文档片段列表。
- `enumerate(result["hits"], start=1)` 可以给命中片段编号。
- `TOP_K=4` 时页面最多展示 4 个命中片段。
- 如果出现 `KeyError: 'hits'`，应优先检查 `ask_with_trace()` 的返回字典。

小改造：

把来源完整路径改成只显示文件名：

```python
source = Path(doc.metadata.get("source")).name
```

### 第 3 组：来源引用展示

已掌握：

- 来源引用服务于可追溯回答。
- 来源引用可以辅助判断模型是否产生幻觉。
- `Path(...).name` 可以去掉目录路径，只保留文件名。
- `TOP_K=2` 时来源引用最多显示 2 条。
- 如果页面报 `NameError: name 'Path' is not defined`，需要补充：

```python
from pathlib import Path
```

### 第 4 组：README 运行说明

已掌握：

- README 要让别人快速、清楚知道项目如何运行。
- `python -m streamlit run ...` 表示用 Streamlit 启动网页应用。
- 如果 README 没有明确写固定返回数量，`TOP_K` 从 4 改成 2 时通常不需要同步修改。
- 找不到 `docs\Day4\streamlit_rag_demo.py` 时，优先检查是否在项目根目录运行命令。
- 示例问题应写在 Day4 运行说明的示例问题代码块里。

## blockers.md：卡点与报错

### 1. 当前虚拟环境缺少 Streamlit

现象：

```text
No module named streamlit
```

类型：依赖环境问题。

最小处理：

```powershell
.\.venv\Scripts\python.exe -m pip install streamlit
```

结果：

```text
Streamlit, version 1.57.0
```

### 2. 后台启动 Streamlit 服务不稳定

现象：

使用 PowerShell 后台启动后，`localhost` 没有稳定响应。

处理方式：

当前阶段先使用前台命令运行：

```powershell
.\.venv\Scripts\python.exe -m streamlit run docs\Day4\streamlit_rag_demo.py
```

原因：

Day4 学习重点是页面功能，不是服务后台保活。

### 3. 页面提交后长时间没有结果的排查顺序

建议顺序：

1. 看终端有没有报错。
2. 看是否正在加载本地 embedding 模型，第一次可能较慢。
3. 看是否卡在 DeepSeek API 调用，检查网络、API key、额度。
4. 回到 Day3 CLI Demo，确认 RAG 核心流程是否还能跑。
5. 如果 CLI 正常、网页异常，再检查 Streamlit 页面代码。
