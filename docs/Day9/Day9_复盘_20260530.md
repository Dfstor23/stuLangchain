# Day9 复盘 - Agent 基础：Function Calling

## 1. 今天完成了什么

### 第 1 步：理解 Function Calling 概念（无代码）

- 理解了 Agent 和 RAG 的核心区别：RAG 流程固定，Agent 由模型自己决定流程。
- 理解了 Function Calling 的核心：你告诉模型有哪些工具可用，模型自己决定是否调用。
- 模型不执行工具，它只输出"我要调用 XXX 工具，参数是 YYY"，执行工具的是代码。

### 第 2 步：最小 Function Calling Demo

- 定义了 `get_current_time` 工具，用 `@tool` 装饰器。
- 用 `bind_tools()` 把工具绑定给 DeepSeek 模型。
- 用 3 个问题测试，模型正确区分了"需要调工具"和"直接回答"。

### 第 3 步：补全工具调用闭环

- 把工具执行结果包装成 `ToolMessage`，送回模型。
- 模型根据工具返回值生成自然语言最终回答。
- 完整闭环：用户提问 → 模型判断 → 调用工具 → 工具返回 → 送回模型 → 最终回答。

## 2. 今天掌握的知识点

- Agent 的核心不是 if-else，而是 `bind_tools()`。判断逻辑在模型内部。
- `@tool` 装饰器把普通函数变成模型可识别的工具，docstring 是模型判断的依据。
- `response.tool_calls` 是一个列表，包含 name、args、id。
- `ToolMessage` 必须带 `tool_call_id`，让模型知道这是哪次调用的结果。
- 对话是 messages 列表，每一轮追加消息，模型看完整历史生成回答。

## 3. 需要记住的补强点

- `response.tool_calls` 是列表不是 bool，空列表 `[]` 在 Python 中是 falsy。
- 工具调用后必须把结果送回模型，否则用户只看到原始数据。
- 工具的 docstring 要写清楚"什么时候该用这个工具"，描述模糊会导致模型不调用。

### 第 1 步检查题（概念）

1. 模型负责决定调工具，代码负责执行：✅
2. "告诉模型有哪些工具"对应什么操作：⚠️（误认为是 if-else，应为 bind_tools）
3. 两个工具选哪个：✅
4. 模型不调工具的排查：✅
5. RAG 做成 Agent 工具的好处：✅

### 第 2 步检查题（代码）

1. @tool 装饰器的作用：✅
2. response.tool_calls 返回什么：⚠️（误认为是 bool，实际是列表）
3. docstring 模糊的影响：✅
4. 所有问题都不调工具的排查：✅
5. 完整闭环流程描述：⚠️（漏了"把工具结果送回模型"这一步）

### 第 3 步检查题（闭环）

1. messages 列表包含哪几种消息类型：⚠️（说成了 str/list，应为 HumanMessage → AIMessage → ToolMessage → AIMessage）
2. ToolMessage 为什么必须带 tool_call_id：✅
3. 模型再次返回 tool_calls 能否处理：✅（正确指出当前无循环逻辑）
4. 工具返回正常但最终回答为空：⚠️（应考虑 tool_call_id 匹配、网络问题等）
5. 新增计算器工具需要改哪里：✅
