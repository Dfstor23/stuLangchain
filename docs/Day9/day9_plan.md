# Day9 任务规划 - Agent 基础：理解 Function Calling

## Day9 总目标

理解 Agent 的核心思想：让大模型自己决定调用什么工具、传什么参数。

目标不是写一个复杂 Agent，而是理解最小 Agent 流程：

```text
用户提问
→ 模型判断是否需要调用工具
→ 如果需要，模型输出工具名称和参数
→ 代码执行工具，拿到结果
→ 把结果交给模型，生成最终回答
```

## 前置知识

Day1-Day8 你已经掌握的 RAG 流程是「固定流程」：

```text
用户提问 → 检索 → 拼接上下文 → 模型回答
```

每次都走同样的步骤，不管问题是什么。

Agent 的核心区别是「模型自己决定流程」：

```text
用户提问 → 模型判断要做什么 → 可能调工具，也可能直接回答
```
