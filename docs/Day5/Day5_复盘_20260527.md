## 2. 检查题与回答记录

### 第 1 步：空检索结果处理

1. 为什么 RAG 页面在 `hits` 为空时，不应该继续展示模型回答？
   - 回答：避免模型没有依据胡乱回答。
   - 点评：正确。更正式的说法是降低模型在缺少检索依据时产生幻觉的风险。

2. `if not result["hits"]`、`st.warning()`、`st.stop()` 分别做什么？
   - 回答：如果没有检索到片段，输出警告信息并提示，然后停止运行。
   - 点评：正确。这里的“停止运行”指停止当前 Streamlit 脚本后续展示逻辑。

3. 如果把 `TOP_K = 4` 改成 `TOP_K = 1`，会怎样？
   - 回答：片段变少，资料不足概率上升。
   - 点评：正确。

4. 如果页面报错 `KeyError: 'hits'`，优先检查什么？
   - 回答：检查 `ask_with_trace()` 函数，因为 `hits` 片段在这个函数中产生输出。
   - 点评：正确。

5. 如果要修改提示文案，应该改哪里？
   - 回答：修改 `st.warning("资料中没有足够信息。")` 这一行。
   - 点评：方向正确。完整改法需要写出新的提示字符串。

### 第 2 步：README 异常提示说明

1. 为什么 README 里要写“资料不足时不强行回答”？
   - 回答：防止模型没有依据回答。
   - 点评：正确。

2. README 这句话和代码中哪几行对应？
   - 回答：`streamlit_rag_demo.py` 中空检索判断、警告提示和停止执行这三行。
   - 点评：正确。

3. 如果 `top_k` 变小，触发“检索不到相关片段”的可能性会怎样？
   - 回答：变大。
   - 点评：正确。

4. 如果 README 写了会提示，但页面实际继续展示回答，优先检查哪里？
   - 回答：检查空检索判断、警告提示和停止执行这几行。
   - 点评：正确。

5. 如何把“避免模型在没有依据时强行回答”说得更正式？
   - 回答：提高模型回答的准确率，降低模型幻觉。
   - 点评：基本正确。更推荐表述为“降低模型在缺少检索依据时产生幻觉的风险”。
