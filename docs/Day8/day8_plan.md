## Day8 学习路径

### 第 1 步：整理正式测试问题集

目标：

把测试问题分成三类：

- 知识库内问题：答案能直接从资料中找到。
- 知识库边界问题：资料中有部分信息，但可能不完整。
- 知识库外问题：资料中没有答案，应该提示资料不足。

为什么要做：

- 只测知识库内问题，无法验证拒答能力。
- 只测知识库外问题，无法验证正常回答能力。
- 分类测试可以更稳定地观察 RAG 质量。

记录文件：

```text
docs/Day8/rag_test_questions.md
```

运行命令：

```powershell
.\.venv\Scripts\python.exe -m streamlit run docs\Day4\streamlit_rag_demo.py
```

预期输出：

- 知识库内问题：能回答，并展示相关命中片段。
- 知识库边界问题：可能只能回答部分内容。
- 知识库外问题：应该提示资料中没有足够信息。

必须掌握的知识点：

- 测试问题集是 RAG 质量优化的基础。
- 知识库边界问题可以帮助发现资料是否不完整。
- 知识库外问题可以帮助验证系统是否会拒答。
