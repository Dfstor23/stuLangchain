# Day8 RAG 测试问题集

## 测试目标

用分类问题集评估当前 RAG Demo 的回答质量。

重点观察：

- 命中片段是否相关。
- 回答是否基于资料。
- 资料不足时是否拒答。
- 哪些问题暴露出知识库内容缺口。

## 当前知识库范围

当前 `data/intro.txt` 主要包含：

- LangChain 的核心能力。
- Prompt 模板的作用。
- RAG 的关键步骤。

- 为什么需要 RAG。
- LangGraph 的基础说明。

## 6. Prompt 优化对比（2026-05-28）

### 优化内容

在 Prompt 中增加一句：`请尽量完整地列出资料中提到的所有相关要点，不要遗漏。`

### 复测问题：RAG 如何减少幻觉？

优化前回答：

```text
RAG 通过注入外部知识，让回答基于真实文档，从而减少幻觉。（只引用片段 1，遗漏"标注来源"）
```

优化后回答：

```text
1. 注入外部知识：让回答基于真实文档（片段 1）。
2. 基于真实文档：回答内容直接来源于真实文档（片段 1）。
3. 标注来源，方便核实：可以标注回答所依据的资料来源（片段 1）。
4. 利用 metadata 检查可靠性：开发者可以知道模型回答依据哪份资料、哪个片段（片段 3）。
```

### 对比结论

- 优化后回答更完整，"标注来源"不再遗漏。
- 模型还额外关联了 metadata 片段，回答更深入。
- 未编造内容，所有要点均有片段支持。
- 这说明 Prompt 优化可以在不改知识库、不改参数的情况下提升回答质量。
