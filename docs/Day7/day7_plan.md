# Day7 任务规划 - RAG 回答质量测试记录

## Day7 总目标

从页面展示回到 RAG 主线，用固定问题集观察当前 Demo 的检索和回答质量。

目标不是新增复杂功能，而是学会记录和判断：

```text
固定问题
-> 查看命中片段
-> 查看模型回答
-> 判断回答是否基于资料
-> 记录问题、命中片段、回答和评价
```

## Day6 已有基础

可以复用：

- `docs/Day4/streamlit_rag_demo.py`
- `docs/Day2/day2.py`
- `data/intro.txt`

当前 Demo 已支持：

- 输入问题。
- 展示模型回答。
- 展示命中片段。
- 展示来源引用。
- 检索不到资料时提示资料不足。

## Day7 学习路径

### 第 1 步：准备 3 个固定测试问题

目标：

先固定问题，避免每次随便问导致无法比较效果。

本次测试问题：

```text
1. RAG 的关键步骤是什么？
2. Prompt 模板在 RAG 中有什么作用？
3. Document.metadata 可以用来做什么？
```

为什么要做：

- 固定问题集可以对比不同参数下的回答质量。
- 这些问题都能在 `data/intro.txt` 中找到依据。
- 测试问题要覆盖流程、Prompt、metadata 三个 RAG 基础点。

运行命令：

```powershell
.\.venv\Scripts\python.exe -m streamlit run docs\Day4\streamlit_rag_demo.py
```

预期输出：

- 每个问题都能看到模型回答。
- 每个问题都能看到命中片段。
- 每个问题都能看到来源引用。

常见问题：

- 回答缺少关键点：检查命中片段是否覆盖完整信息。
- 命中片段不相关：后续可以对比 `top_k`、`chunk_size`、`chunk_overlap`。
- 页面报 API key 问题：检查 `.env` 中的 `DEEPSEEK_API_KEY`。

必须掌握的知识点：

- 固定问题集是 RAG 质量评估的基础。
- RAG 不是只看回答好不好，还要看命中片段是否支持回答。
- 评价回答时要关注“是否基于资料”和“是否遗漏关键点”。

### 第 2 步：记录 3 个问题的测试结果

目标：

把每个问题的命中片段、模型回答和简单评价写入测试记录。

记录文件：

```text
docs/Day7/rag_quality_test_20260527.md
```

### 第 3 步：根据测试结果找一个最小优化方向

目标：

只选一个最明显的问题，例如：

- 某个问题命中片段不完整。
- 某个回答缺少来源中的关键点。
- 某个参数可能需要调整。

暂时不做：

- 新增 Agent。
- 上传文件。
- 多格式解析。
- 大规模自动评测。

## 下一次只推进的最小任务

先运行页面，依次输入 3 个固定问题，并把每个问题的结果填入测试记录。
